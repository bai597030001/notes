    6.8 NULL 值处理
       为什么要对NULL值进行处理
       因为NULL值和任何值做运算结果都是NULL
       换一种年薪计算的方式
          月薪*12 + 月薪*12*提成/100  
       select  salary*12,
           salary*12*(1+commission_pct/100)
               from s_emp;      
       nvl(par1,par2)
           当par1的值为NULL时 就返回par2
           的值 如果par1不为NULL就返回
           par1本身的值。 
       select  salary*12,
           nvl(salary*12*(1+commission_pct/100),0)
               from s_emp;
       /* NULL 值要尽早处理 */        
       select  salary*12,
           salary*12*(1+nvl(commission_pct,0)/100)
               from s_emp;        
       显示s_emp 表中 salary first_name 
           manager_id 如果manager_id 是
           NULL就显示成 -1 
       select  salary,first_name,
           nvl(manager_id,-1)
           from s_emp;
       nvl 中两个参数的类型要保持一致

 七.where 子句
    7.1 where子句的作用
    限制表中的行数据返回 符合where条件的

    from 表名  where 条件;
    /* 永真条件 */

    from 表名  where 1=2;

    7.7 逻辑条件连接符
    and    ca and cb 
        ca 和 cb 同为真时
       找出工资在[1400,2500] 之间的
       select id,salary  
           from  s_emp 
               where salary>=1400 and 
               salary <= 2500; 
    or     ca or  cb 
        ca 和cb只要有一个为真即可 
        部门编号是31或者是 32 或者是50
        的员工 列出 id first_name dept_id
        select  id,first_name,dept_id 
            from s_emp 
                where dept_id=31 or 
                dept_id=32 or dept_id=50; 
       
    not   对立面问题
       =      !=
       >      <=
       <      >=
       between a and b  not between a and b
       in     not in(注意NULL值)
       like   not like 
       is null  is not null 
    找出提供不是NULL的员工  列出
       id  first_name  commission_pct 
    select  id,first_name,commission_pct
       from  s_emp where commission_pct
       is not null;
