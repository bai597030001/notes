在Cta项目中，用到了该线程池。发现信号SIGUSR1触发任务后并没有执行对应的Task类的子类任务，
观察后发现其原因是没有调用子类的run()函数，转而调用了基类(即Task)的run()函数。

进一步分析后得出结论：
	主线程中添加子类指针到线程池任务队列时，可能会出现刚添加子类指针到任务队列，还未执行该任务，
	此时主线程中已经执行到了右括号(即子类指针指向的对象随着作用域的退出而析构)，此后再次执行该
	任务时，只能调用到基类(即Task)的方法，也就是run()函数。
	
解决方法：
	每次添加任务到队列时，在堆区动态创建子类，并通过基类指针指向该子类，随后将基类指针加入任务队列，
	这样的话加入到队列的指针不会随着作用于的消失而失效，可实现多态调用。唯一需要注意的是当从任务队列
	取出任务执行完毕后，需要删除(delete)该指针对应的子类对象。
